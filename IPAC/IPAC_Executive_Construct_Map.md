# IPAC Scenario — Executive Construct Map

*Which behaviours in the exercise map to which executive constructs, and where the evidence comes from. This is an **evidence map for human-anchored scoring**, not an automated trait engine — see the note at the end before any of it carries weight in a decision.*

**Evidence sources**
- **INQ** — questions to the AI assistant (the inquiry trail)
- **MEMO** — the written deliverable(s) (Task 1 board paper, Task 2 reply)
- **PACE** — timing, sequencing, when they commit
- **REV** — revision / editing behaviour
- **TRIAGE** — in-tray prioritisation actions *(dynamic in-basket mode only)*
- **IM** — instant-message / email replies under load *(dynamic mode only)*

Constructs marked **\*** become observable only if you run the email-inbox / IM interruption version. The static scenario still yields everything else.

## Before this carries weight

- **It's an evidence map, not a trait score.** Each row needs behavioural anchors (a described 1–5 scale), rater calibration, and reliability checks before it informs a decision. The defensible claim remains "evidence of this construct in this exercise," not a validated personality score.
- **Score judgement, not velocity.** The TRIAGE and IM streams especially must be scored on the *quality of the decisions* — what they protected, deferred, delegated — never on speed or volume, which proxies for language fluency, age and neurodivergence.
- **Run an adverse-impact check** on any construct before it counts, particularly the dynamic-mode ones.
